# AI-Driven Marketing Intelligence & Campaign Automation

## 📌 Project Overview
This prototype is a Python-based Artificial Intelligence system designed for NETSCOUT Systems. Its primary objective is to perform AI-driven marketing intelligence to position NETSCOUT’s Omnis CyberStream and Omnis Cyber Intelligence NDR platforms against Darktrace's Cyber AI Platform. 

The system utilizes a Retrieval-Augmented Generation (RAG) architecture to extract insights from technical whitepapers and data sheets, analyzes market sentiment, and automatically generates targeted marketing content.

---

## 🏗️ System Architecture
The system is built as a sequential pipeline consisting of four main phases:

1.  **Document Collection & Extraction:** Parses raw PDF documentation (whitepapers, data sheets) to extract and clean unstructured text.
2.  **Knowledge Base Creation (RAG):** Splits the extracted text into overlapping semantic chunks, generates vector embeddings, and stores them in a local Vector Database for semantic search.
3.  **Market Intelligence (Traditional ML & Sentiment):** Uses Natural Language Processing (NLP) to gauge the sentiment of simulated social media discussions, and applies K-Means clustering (Traditional ML) to categorize posts into distinct topics.
4.  **AI Insight & Content Generation:** Queries the Vector Database for competitive facts, feeds them into an LLM (Google Gemini), and generates structured competitive insights and ready-to-publish marketing collateral.
5.  **Marketing Simulation:** A Python workflow that simulates the automated publishing of generated assets to social channels and tracks mock engagement metrics.

---

## 🛠️ Tools and Libraries Used
* **PDF Extraction:** `pdfplumber` (Chosen for its superior handling of multi-column layouts and tables compared to basic PyPDF).
* **AI Framework & Orchestration:** `langchain`, `langchain-text-splitters`, `langchain-community`
* **Vector Database:** `chromadb` (An efficient, open-source local vector database).
* **Embeddings:** `langchain-huggingface` / `sentence-transformers` (Model: `all-MiniLM-L6-v2`).
* **Large Language Model (LLM):** `google-generativeai` (Google Gemini 1.5 Flash).
* **Traditional Machine Learning:** `scikit-learn` (K-Means Clustering, TF-IDF Vectorization).
* **Sentiment Analysis & Data Handling:** `textblob`, `pandas`

---

## 🧠 Key Design Decisions

1.  **`pdfplumber` over `PyPDF2`:** The provided NETSCOUT and Darktrace documentation contained complex multi-column layouts and tabular SKU data. `pdfplumber` was selected to maintain the geometric integrity of the text, preventing sentences across columns from being mashed together, which would confuse the LLM.
2.  **Local HuggingFace Embeddings:** Instead of using paid API calls for embedding generation (e.g., OpenAI embeddings), a local HuggingFace MiniLM model was utilized. This reduces API dependency, increases processing speed, and costs nothing to run.
3.  **Simulated Social Media Dataset:** Real-time extraction from X (Twitter) and LinkedIn APIs is highly restricted, often requiring paid developer tiers and complex authentication. To demonstrate the machine learning and sentiment analysis capabilities reliably, a simulated but highly realistic dataset of market discussions was utilized.
4.  **Google Colab Native Design:** The code was structured to run seamlessly in Google Colab, utilizing `google.colab.files` for direct file uploading to bypass complex local environment pathing issues.

## 🚀 Instructions to Run the Solution

This project is designed to run in **Google Colab**. 

### Prerequisites:
1. You must have the source PDFs downloaded to your local computer.
2. You need a free Google Gemini API Key (Available at: https://aistudio.google.com/app/apikey).

### Execution Steps:
1. Open the provided `.ipynb` notebook file in Google Colab.
2. Run **Cell 1** to install all required dependencies.
3. Run **Cell 2.1**. A "Choose Files" button will appear. Click it and upload all the provided NETSCOUT and Darktrace PDFs.
4. Run **Cells 2.2 through 2.4** to extract the text and build the ChromaDB Vector Database.
5. Run **Cells 3.1 through 3.3** to execute the Sentiment Analysis and K-Means Topic Clustering.
6. In **Cell 4.1**, replace `"YOUR_API_KEY_HERE"` with your actual Gemini API Key, then run the cell.
7. Run **Cells 4.2 through 4.4** to generate the Competitive Insights and Marketing Content via the LLM.
8. Run **Cell 5.1** to execute the marketing automation publishing simulation.
